# 芋道商城 uni-app 组件说明文档

> 说明：本文档根据你截图中的 `components` 目录命名整理，适合用于快速理解每个组件的大概作用。具体参数、字段和样式仍要以你项目里每个组件目录下的 `index.vue`、`props`、`scss` 为准。

---

## 1. 首页装修 / 页面搭建类组件

这些组件通常和商城装修 JSON 关系最密切，首页、我的页面、活动页大多会用到。

| 组件名 | 中文理解 | 主要作用 | 常见使用位置 | 你是否需要重点改 |
|---|---|---|---|---|
| `s-layout` | 页面布局容器 | 承载页面装修组件，统一处理页面背景、底部导航、状态栏、页面容器等 | 首页、分类页、我的页、装修页 | 高 |
| `s-custom-navbar` | 自定义导航栏 | 控制顶部导航栏样式、标题、返回按钮、沉浸式效果 | 首页顶部、详情页顶部、我的页顶部 | 中 |
| `s-statusbar` | 状态栏占位 | 适配手机顶部状态栏高度，防止内容被刘海屏/状态栏遮挡 | 顶部导航区域 | 低 |
| `s-tabbar` | 底部导航栏 | 首页、分类、购物车、我的等底部 Tab 切换 | 全局底部 | 中 |
| `s-search-block` | 搜索栏组件 | 首页/分类页的搜索框，支持 placeholder、搜索入口、样式配置 | 首页顶部、分类页顶部 | 中 |
| `s-title-block` | 标题栏组件 | 模块标题，例如“热门服务”“新人专区”“服务保障” | 首页各模块标题 | 中 |
| `s-notice-block` | 公告栏组件 | 滚动公告/提示文案，例如“下单前请联系客服确认需求” | 首页 Banner 下方 | 中 |
| `s-line-block` | 分割线组件 | 页面分割线、模块间隔线 | 各模块之间 | 低 |
| `s-richtext-block` | 富文本组件 | 展示后台配置的富文本内容，例如规则说明、活动说明 | 商品详情、活动页、说明页 | 中 |
| `s-video-block` | 视频组件 | 展示视频内容，可能支持封面、播放地址 | 首页、活动页、详情页 | 低 |
| `s-popup-image` | 弹窗图片 | 进入页面弹出的活动广告图 | 首页、活动页 | 低 |
| `s-float-menu` | 悬浮菜单 | 页面右侧悬浮按钮，例如客服、返回顶部、分享 | 首页、详情页 | 中 |
| `s-menu-button` | 菜单按钮 | 单个菜单按钮组件，可能被宫格/列表复用 | 菜单入口 | 中 |
| `s-menu-grid` | 宫格菜单 | 首页分类入口，例如新人特惠、游戏陪玩、护航、趣味单 | 首页分类入口 | 高 |
| `s-menu-list` | 列表菜单 | 纵向菜单列表，例如服务规则、联系客服、售后说明 | 我的页、设置页、帮助页 | 高 |
| `s-menu-tools` | 工具菜单 | 常用工具入口集合，例如客服、订单、收藏、优惠券等 | 我的页、工具区 | 中 |

## 2. 图片 / 广告 / 热区类组件

这些组件负责首页视觉效果。你现在的黑红电竞风，主要靠这些组件承载。

| 组件名 | 中文理解 | 主要作用 | 常见使用位置 | 你是否需要重点改 |
|---|---|---|---|---|
| `s-image-banner` | 轮播 Banner | 首页顶部轮播图，展示活动、品牌、主推服务 | 首页首屏 | 高 |
| `s-image-block` | 单图展示 | 展示一张横幅图，例如服务流程图、保障图、新人专区图 | 首页中部、底部 | 高 |
| `s-image-cube` | 图片魔方 | 多图拼接广告位，支持一行多图、不规则布局 | 首页活动区 | 中 |
| `s-hotzone-block` | 图片热区 | 在一张图片上配置多个可点击区域 | 活动页、专题页 | 中 |

### 对你的项目建议

你的首页想做成 Delta 先锋俱乐部风格，建议主要用：

```text
s-image-banner：顶部品牌轮播图
s-image-block：新人专区图、服务流程图、服务保障图
s-image-cube：后期做活动专区/多套餐入口
s-hotzone-block：后期做一张大图多个点击区域
```

## 3. 商品展示类组件

这些组件决定首页像不像“商城”。你现在首页不像卖商品，重点就要改这些。

| 组件名 | 中文理解 | 主要作用 | 常见使用位置 | 你是否需要重点改 |
|---|---|---|---|---|
| `s-goods-card` | 商品卡片 | 单个商品卡片，通常包含商品图、名称、价格、销量、按钮 | 商品列表、首页推荐 | 高 |
| `s-goods-item` | 商品项 | 商品基础项，可能是更底层的商品展示组件 | 商品列表内部 | 高 |
| `s-goods-column` | 商品双列/多列布局 | 控制商品瀑布流、双列商品展示 | 首页商品区、分类商品列表 | 高 |
| `s-goods-scroll` | 横向滚动商品 | 一行横向滑动商品列表 | 秒杀区、新人专区、推荐区 | 中 |
| `s-goods-shelves` | 商品货架 | 按后台装修配置展示一组商品，可能支持列表/双列/横滑 | 首页热门商品、新人专区 | 高 |
| `s-discount-list` | 折扣商品列表 | 展示折扣/促销商品 | 活动页、首页促销区 | 中 |

### 对你的项目建议

你最应该优先修改：

```text
s-goods-card
s-goods-item
s-goods-column
s-goods-shelves
```

目标是把默认商品卡片改成：

```text
商品封面 1:1
商品名称两行省略
红色价格
服务标签：绿色组队 / 客服确认 / 售后有据
按钮：立即下单
卡片圆角和轻微阴影
```

## 4. 用户中心 / 我的页面类组件

这些组件主要用于“我的页面”。你现在觉得我的页面丑，重点看这些组件。

| 组件名 | 中文理解 | 主要作用 | 常见使用位置 | 你是否需要重点改 |
|---|---|---|---|---|
| `s-user-card` | 用户信息卡 | 显示头像、昵称、登录状态、会员信息等 | 我的页面顶部 | 高 |
| `s-order-card` | 订单入口卡 | 显示待付款、待发货、待评价、售后、全部订单等 | 我的页面订单区 | 高 |
| `s-wallet-card` | 钱包卡片 | 显示余额、积分、优惠券、钱包入口 | 我的页面资产区 | 低/可隐藏 |
| `s-coupon-card` | 优惠券卡片 | 单张优惠券展示 | 优惠券列表、领券中心 | 低 |
| `s-coupon-list` | 优惠券列表 | 展示用户已有优惠券或可领取优惠券 | 我的优惠券、领券页 | 低 |
| `s-coupon-get` | 领取优惠券 | 领取优惠券按钮或弹窗 | 活动页、商品详情页 | 低 |
| `s-coupon-select` | 选择优惠券 | 下单页选择可用优惠券 | 确认订单页 | 低 |
| `s-coupon-block` | 优惠券装修块 | 首页/活动页展示优惠券模块 | 首页、活动页 | 低 |

### 对你的项目建议

第一版游戏服务商城可以弱化这些：

```text
余额
积分
优惠券
领券中心
```

更适合你的“我的页面”结构是：

```text
会员中心头图
用户信息卡
我的订单
服务中心：联系客服 / 服务规则 / 售后说明 / 服务保障
服务流程图
保障承诺图
```

## 5. 下单 / SKU / 购买流程类组件

这些组件和商品详情、下单规格选择有关。

| 组件名 | 中文理解 | 主要作用 | 常见使用位置 | 你是否需要重点改 |
|---|---|---|---|---|
| `s-select-sku` | 普通商品规格选择 | 选择商品规格、数量、价格，加入服务清单或立即购买 | 商品详情页 | 高 |
| `s-select-seckill-sku` | 秒杀规格选择 | 秒杀商品的规格选择弹窗 | 秒杀商品详情 | 低 |
| `s-select-groupon-sku` | 拼团规格选择 | 拼团商品的规格选择弹窗 | 拼团商品详情 | 低 |
| `s-address-item` | 地址项 | 收货地址卡片，展示姓名、电话、地址 | 确认订单、地址列表 | 低/可能不用 |
| `s-uploader` | 上传组件 | 上传图片/文件，例如售后凭证、评价图片 | 售后、评价、资料上传 | 中 |

### 对你的项目建议

因为你卖的是游戏服务，不是实物，`s-select-sku` 很重要。你可以把 SKU 设计成：

```text
服务类型：娱乐 / 武将 / 武圣 / 五虎上将
服务时长：30分钟 / 1小时 / 2小时
区服：微信 / QQ / PC / 双端
是否加急：普通 / 加急
```

`地址组件` 第一版可以不用，或者后期改成“游戏账号信息/联系方式/备注需求”。

## 8. 组件和装修 JSON 的大致对应关系

你后台装修 JSON 里面的组件名，前端通常会映射到这些 `s-xxx` 组件。不同版本命名可能不完全一致，但大致对应如下：

| 装修 JSON 里的 id | 前端组件可能对应 | 说明 |
|---|---|---|
| `SearchBar` | `s-search-block` | 搜索栏 |
| `Carousel` | `s-image-banner` | 轮播图 |
| `ImageBar` | `s-image-block` | 单图广告 |
| `ImageCube` | `s-image-cube` | 图片魔方 |
| `MenuGrid` | `s-menu-grid` | 宫格菜单 |
| `MenuList` | `s-menu-list` | 列表菜单 |
| `NoticeBar` | `s-notice-block` | 公告栏 |
| `TitleBar` | `s-title-block` | 标题栏 |
| `ProductList` | `s-goods-shelves` / `s-goods-card` / `s-goods-column` | 商品列表/商品货架 |
| `UserCard` | `s-user-card` | 用户卡片 |
| `UserOrder` | `s-order-card` | 订单卡片 |
| `UserCoupon` | `s-coupon-block` / `s-coupon-list` | 优惠券模块 |
| `RichText` | `s-richtext-block` | 富文本 |
| `Video` | `s-video-block` | 视频组件 |

## 9. Delta 先锋俱乐部项目优先修改顺序

如果你想把页面变好看，建议按这个顺序改，不要一口气全改。

### 第一优先级：商城观感

```text
1. s-goods-card
2. s-goods-item
3. s-goods-column
4. s-goods-shelves
5. s-menu-grid
```

目的：让首页像一个真正能下单的商城。

### 第二优先级：品牌风格

```text
1. s-image-banner
2. s-image-block
3. s-title-block
4. s-notice-block
5. s-custom-navbar
```

目的：统一黑红电竞、战术科技风。

### 第三优先级：我的页面

```text
1. s-user-card
2. s-order-card
3. s-menu-list
4. s-menu-tools
5. s-empty
```

目的：让“我的页面”不像默认模板。

### 第四优先级：下单体验

```text
1. s-select-sku
2. s-address-item
3. s-uploader
4. s-auth-modal
```

目的：让用户能顺畅选择套餐、提交需求、查看订单。

## 10. 给 AI 开发时可以这样说

```text
你是一个熟悉 uni-app、Vue3、芋道商城 yudao-mall-uniapp 的前端工程师。

我现在要优化 Delta 先锋俱乐部商城首页和我的页面样式。项目组件目录下有很多 s-xxx 组件，例如 s-goods-card、s-goods-shelves、s-menu-grid、s-image-banner、s-image-block、s-user-card、s-order-card 等。

请你先不要改后端，不要改数据库结构，只修改前端组件样式。

优先修改：
1. s-goods-card：商品卡片改成更像商城，白底卡片、圆角、商品图 1:1、标题两行、红色价格、立即下单按钮、服务标签。

5. s-notice-block：公告栏改成白色圆角卡片。
6. s-user-card / s-order-card：我的页面改成更符合 Delta 先锋俱乐部风格。

页面主体背景使用 #f3f4f6，主色使用 #ff3b30，卡片背景 #ffffff，品牌图片保持黑红电竞风。

完成后请告诉我修改了哪些文件，每个文件改了什么。
```

## 11. 总结

你这批组件可以分成 6 类：

```text
页面装修类：s-layout / s-custom-navbar / s-tabbar / s-title-block / s-notice-block
图片展示类：s-image-banner / s-image-block / s-image-cube / s-hotzone-block
商品展示类：s-goods-card / s-goods-item / s-goods-column / s-goods-scroll / s-goods-shelves
用户中心类：s-user-card / s-order-card / s-wallet-card / s-coupon-xxx
购买流程类：s-select-sku / s-address-item / s-uploader
营销活动类：s-seckill-block / s-groupon-block / s-point-block / s-live-block
```

对你的项目来说，最重要的不是把所有组件都改一遍，而是先改这几个：

```text
s-goods-card
s-goods-shelves
s-menu-grid
s-image-banner
s-image-block
s-user-card
s-order-card
```

这几个改完，你的首页和我的页面就会明显不像默认模板了。
